Program Objective

Deliver a Bengaluru-grade flood intelligence system that transforms raw camera images into reliable depth estimates, 5-level alert signals, and municipal action triggers with traceable cloud operations.

City Safety Objective

Reduce urban flood response time through automated depth estimation and clear emergency action recommendations.

Engineering Objective

Use one standard event contract and one deterministic execution path across API, queue workers, and serving adapters.

Data Objective

Enforce strict payload validation, manifest-based labeling, and corruption quarantine to protect model quality.

Ops Objective

Add retry, observability, and dead-letter routing so failures are durable, traceable, and recoverable.

Approach Taken

Implementation strategy used in this workspace to move from PoC fragments toward enterprise-ready, modular architecture.

A. Contract First `src/event_contract.py` defines canonical `FloodEvent`, `FloodResultEvent`, and failure envelope.
B. Single Pipeline `src/pipeline.py` is the shared core: decode → water gate → infer → aggregate → color/action map.
C. Shared Middleware `src/middleware/retry.py` + `src/middleware/observability.py` apply common reliability and trace logging.
D. Durable Failures `src/dlq.py` publishes permanent failures to Redis DLQ (or JSONL fallback) with standard error envelope.
