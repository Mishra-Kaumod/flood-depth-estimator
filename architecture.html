🌊 Bengaluru Flood Depth Estimator

End-to-end system architecture — 5-stage inference pipeline with optional Gemini Vision enhancement

📷 RGB Image Input
Stage 1 SegFormer WaterRegionDetector (classical CV) Always Classical

Segments flood water pixels using color-space analysis, edge detection, and morphological operations. No pre-trained neural model required at runtime.

Output
water_mask[H,W] · water_coverage_pct: float
Stage 2 YOLOv8 yolov8n.pt or contour proxy Always Classical

Detects real-world reference objects (cars, people, motorcycles) for scale calibration. Falls back to contour-based proxy when yolov8n.pt is absent.

Output
references: List[ReferenceObject] · ref_backend: str
Stage 3 Depth Anything V2 Dense depth proxy map Classical fallback ✨ Gemini-enhanced

Produces a dense per-pixel depth proxy map using luminance inversion, vertical gradient prior, and texture energy. With Gemini: Vision API analyzes the image for a scene-level depth estimate that refines the proxy.

🔑 Requires GEMINI_API_KEY — optional. Classical proxy always runs.
Output
dense_depth_map[H,W]: float32 · gemini_max_depth_cm (optional)
Stage 4 Fusion Engine feature-fusion-v1 Classical fallback ✨ Gemini-enhanced

Combines water coverage, reference object submersion, and depth-map statistics into structured features. With Gemini: fused depth estimate blends Gemini's scene understanding (70%) with sensor math (30%).

🔑 Gemini fuses visual context with structured features. Falls back to math-only.
Output
features: Dict[str, float] · gemini_fused_depth_cm (optional)
Stage 5 Calibration / Severity Model calibration-v1 Classical fallback ✨ Gemini-enhanced

Converts structured features into final depth (cm), confidence, severity band, and municipal action trigger. With Gemini: cross-validates and calibrates the final severity classification with a rationale.

🔑 Gemini returns calibrated depth, severity, next action, and rationale.
Output
depth_cm · confidence · severity: SAFE→CRITICAL · action_trigger · rationale
